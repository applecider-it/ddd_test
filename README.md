# DDDの試験的な実装

あくまで実装例で、利用は非推奨です。

通常のアプリケーションでは、`laravel_test`, `rails_test`のような、MVCS（Sは、カテゴリーごとにディレクトリを分けたサービスクラス）が無難。

DDD、クリーンアーキテクチャのエッセンス（ドメインモデル、ユビキタス言語）は、railsである程度実現できていて、あとは、ドメイン境界や、ビジネスロジック置き場がないことをフォローすればいいだけなので、MCVにSをつければいいだけになる。

DDD、クリーンアーキテクチャは、何が問題かは、実際に、実装例を見てもらえばわかると思う。
